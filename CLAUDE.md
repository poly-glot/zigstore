# zigstore: Project Contract

`zigstore` is a reusable, embeddable **and** networked ordered byte-key/byte-value record
store: WAL-durable paged storage, a B+Tree over opaque `[]const u8` keys, an epoll reactor,
a binary-protocol framing/TLV codec, and reflection-driven client codegen. It is extracted
from `dmozdb` to be consumed as a `build.zig.zon` dependency, public and semver-tagged at
`github.com/poly-glot/zigstore`. This file loads every session. The enforceable conventions
live in [`.claude/rules/`](.claude/rules/). Read them.

## Core principles

- **Simplicity first.** The smallest change that correctly solves the problem wins. Justify
  added machinery against the current scale, not a hypothetical one.
- **No laziness, no band-aids.** Fix root causes, not symptoms. If you patch a symptom to
  unblock, say so and name the real fix.
- **Declarative, testable, maintainable.** Senior-level code: clear names, focused units, no
  clever one-liners, behavior covered by tests.

## The boundary is the product

`zigstore` knows nothing about any consumer's records, hierarchy, slugs, or statuses. The
one unforgivable regression is a dependency cycle back to a consumer.

- **No engine file may import or reference an app file or type.** The dependency graph points
  one way: apps depend on `zigstore`, never the reverse.
- **Comptime data plane + runtime seams.** The typed storage layer is generated by
  `Engine(schema)` at comptime; the dynamic seams (request dispatch, recover hooks, periodic
  workers, the snapshot host) are runtime interfaces and callbacks the consumer supplies.
  Keep that split — comptime at the dynamic seams would over-couple.

## Verify before "done" (non-negotiable)

A change is done only when the gate passes and you have cited the evidence.

- **`zig build test`** (exit 0) on pinned **Zig 0.15.2**.
- **`zig build -Dcpu=baseline`** must compile (the OKE Ampere A1 target).
- The on-disk WAL/file path needs Linux `O_DIRECT`/`fallocate`, so the full gate runs in the
  **Linux devcontainer**, not the macOS host. The in-memory surface builds and tests on the
  host; anything touching the file backend does not.
- Cite the evidence in your summary. The Stop hook runs `zig build test` and blocks turn-end
  on failure.

## Source of truth

The pinned toolchain lives in **`build.zig.zon`** (`minimum_zig_version`) and the
devcontainer's `post-create.sh`. Docs describe it; they never override it. When a doc
disagrees, fix the doc.

## Comment policy — OVERRIDES the global no-comment rule, this repo only

Because `zigstore` is a public library, its **exported API surface is documented**:

- **DO** write `///` and `//!` doc comments on **exported declarations** — the schema-plug
  API surface — so Zig `autodoc`, IDE hovers, and third-party discoverability work.
- **NEVER** comment internal/private code. Intent through naming and structure; invariants
  encoded as named tests. No inline WHY comments, no section dividers, no step labels.

See [`zig.md`](.claude/rules/zig.md) for the exact rule.

## Versioning

semver, first cut `v1.0.0`. MAJOR for any wire-format or public-API break. MINOR is additive
only — extend ops with optional trailing bytes; never fork an op number. **Never move a
published tag** (the dependency hash is content-addressed). The commit that changes the wire
format also regenerates the client, in the same commit.
